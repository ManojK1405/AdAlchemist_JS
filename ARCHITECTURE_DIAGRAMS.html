3. Sequence Diagram (Long-Running Operation Pattern)

This diagram highlights the critical LRO (Long Running Operation) architecture used for heavy AI tasks like Veo Video generation. It shows the client polling mechanism preventing server timeouts.

sequenceDiagram autonumber actor User as Client (React) participant API as Express Server participant DB as Postgres DB participant Veo as Veo 3.1 AI User->>API: POST /generate-video (Project ID) API->>API: Validate Credits & Inputs API->>DB: Create GenerationJob (Status: PENDING) API->>Veo: Submit Request Veo-->>API: Returns Operation ID API->>DB: Update Job (Status: PROCESSING, opId) API-->>User: 202 Accepted (Job ID) loop Status Polling (Every 10s) User->>API: GET /job-status/:id API->>DB: Fetch Job alt is Processing API->>Veo: GET /operation/:opId Veo-->>API: Status: Running API-->>User: 202 Processing else is Complete API->>Veo: GET /operation/:opId Veo-->>API: Status: Done (Video URL) API->>DB: Update Project & Job (COMPLETED) API-->>User: 200 OK (Video URL) end end
